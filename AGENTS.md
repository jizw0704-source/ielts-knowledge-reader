# IELTS Knowledge Reader 项目规则

## 规则优先级

1. `AGENTS.md` 是 Codex 执行规则文件，优先级最高。
2. `docs/PROJECT_RULES.md` 是产品规则和需求细则文件。
3. 如果 `AGENTS.md` 与 `docs/PROJECT_RULES.md` 出现冲突，以 `AGENTS.md` 为准。
4. 如果产品需求需要调整，应先更新 `docs/PROJECT_RULES.md`，再进入代码开发。
5. 如果执行边界、目录边界、跨项目修改限制发生冲突，必须以 `AGENTS.md` 为准。

## 项目定位

IELTS Knowledge Reader 是面向雅思阅读能力提升的英文知识阅读器，不是雅思真题售卖平台。
项目的核心价值是通过原创 IELTS-style 英文文章、点词释义、生词本、摘要、标签和阅读记录，帮助用户在真实语境中积累词汇和知识。

## 第一版目标

完成手机端 H5 MVP，并打通以下核心闭环：

今日文章 -> 阅读文章 -> 点词释义 -> 加入生词本 -> 查看生词本 -> 查看文章库

## 当前阶段

当前处于项目规则和 MVP 开发的协同阶段。
业务代码开发前，必须先确认 `AGENTS.md` 与 `docs/PROJECT_RULES.md` 已被读取并理解。

## 严格禁止

1. 只允许修改本项目目录 `Y:\codex\ielts-knowledge-reader` 内的文件。
2. 不允许修改 `Y:\codex` 下其他项目文件。
3. 不允许跨项目移动、删除、重命名文件。
4. 不允许在 `Y:\codex` 根目录创建项目文件。
5. 不要大规模重构。
6. 不要新增无关功能。
7. 如需新增依赖，必须先说明原因。
8. 如需删除文件，必须先获得用户确认。

## 每次任务前必须确认

在修改任何文件前，必须先说明：

1. 当前工作目录
2. 目标项目目录
3. 本次计划修改哪些文件
4. 是否会影响其他项目

每次任务开始前，还必须先读取：

1. `AGENTS.md`
2. `docs/PROJECT_RULES.md`

## 每次任务后必须说明

在完成修改后，必须说明：

1. 实际修改了哪些文件
2. 是否只修改了本项目
3. 如何在本地运行和测试
4. 如果无法运行测试，要说明原因

## 版权边界

不得内置、复制、改写或售卖真实 IELTS、Cambridge IELTS、剑桥雅思真题、官方样题等受版权保护内容。
MVP 阶段只能使用原创 IELTS-style 英文文章、公开授权文章，或用户自行导入内容。
项目中的样例文章必须是原创 IELTS-style 英文文章，不能来自真实雅思材料。

## 技术边界

第一版只做手机端 H5，使用 HTML、CSS、JavaScript 和 `localStorage`。
暂不做 App、微信小程序、后端、登录、支付、AI 接口、自动采集或真实雅思原文库。

## 本项目边界

项目目录为：

`Y:\codex\ielts-knowledge-reader`

除非用户明确要求，否则不得修改此目录以外的任何文件。
