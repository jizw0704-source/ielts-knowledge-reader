## 2. 每次修改后的必测主链路

每次修改 index.html、style.css、script.js 后，必须人工验收：

1. 首页“开始阅读”可点击；
2. 点击后能进入阅读页；
3. 底部导航“今日 / 文章库 / 生词本”可切换；
4. 文章库可展示文章；
5. 文章库标签筛选可用；
6. 点击文章可进入阅读页；
7. 阅读页点击英文单词可弹出释义；
8. 释义弹窗内容清晰；
9. 释义弹窗关闭按钮可点击；
10. “加入生词本”按钮可点击；
11. 生词可写入 localStorage；
12. 刷新后生词仍保留；
13. 生词本可删除单词；
14. 熟悉程度可切换；
15. 微信内置浏览器可正常使用。

## 3. 高风险修改限制

1. 修改 UI / 颜色前必须先新建分支；
2. 禁止一次性大范围修改 style.css；
3. 禁止使用 [class*="xxx"] 这类宽泛选择器影响交互元素；
4. 禁止随意修改 z-index、pointer-events、position、display、visibility、filter、backdrop-filter；
5. 禁止在未确认功能稳定前继续叠加新优化；
6. 修改后必须完成回归测试。
